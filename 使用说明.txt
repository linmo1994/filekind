================================================================================
  filekind — 按项目自动整理资料（使用说明）
================================================================================

【这个工具能帮您做什么】

  您手头有一批项目资料（PDF、Word、Excel、图片等），文件名五花八门，
  分散在不同文件夹里。本工具可以：

  · 根据「项目清单 Excel」和文件名、内容，自动判断每个文件属于哪个项目
  · 把文件复制到「已整理」目录下，按项目名称分文件夹存放
  · 原文件仍保留在「待整理」里，不会误删
  · 全部在本地电脑运行，资料不上传网络

  适合场景：国高申报、研发项目归档、测试报告/规格书/发票等按项目归类。

  典型效果：
    待整理/202301_CV950D4-B42-12 Specification.pdf
      → 已整理/CV950D4基于Android R的…系统软件/202301_CV950D4-B42-12 Specification.pdf

--------------------------------------------------------------------------------
【使用前准备】

  1. 使用打包好的文件夹（推荐）
     打开 dist/filekind/ 整个文件夹，不要只复制单个程序。

  2. 文件夹里应包含：
     · filekind / filekind.exe   主程序
     · 双击可整理文件.command   （Mac 一键整理，推荐）
     · run-filekind.bat          （Windows 一键整理，推荐）
     · _internal/                （Windows 依赖目录，必须与 exe 同目录）
     · projects.yaml             默认配置（一般不用改）
     · 使用说明.txt              本文件
     · 待整理/                   放入待分类的文件
     · 已整理/                   整理结果输出位置
     · 项目清单/                 可选，专门放项目清单 Excel

  3. 准备「项目清单 Excel」
     · 表格需包含「项目编号」「项目名称」等列（与现有国高/版型清单格式一致）
     · 文件名随意，例如：国高资料清单2026 版型清单.xlsx
     · 放入「待整理/」或「项目清单/」即可，不必修改配置文件

  4. 首次运行 Mac 提示「无法打开」时
     · 在 filekind 文件夹上右键 → 打开，或按系统提示允许运行
     · 双击 .command 脚本会自动处理常见权限问题

--------------------------------------------------------------------------------
【推荐操作步骤（文员日常）】

  第一步：放入文件
    · 把需要整理的文件复制到「待整理/」文件夹（可含子文件夹）
    · 把项目清单 Excel 放入「待整理/」或「项目清单/」

  第二步：双击运行
    · Mac：双击「双击可整理文件.command」
    · Windows：双击「run-filekind.bat」
    · 黑色窗口会打开并显示进度（属正常现象，请勿中途关闭）

  第三步：选择项目清单（如需要）
    · 若只发现一份合格清单 → 自动使用，无需操作
    · 若发现多份清单 → 按提示输入序号（1、2、3…）选择
    · 若未自动发现 → 会弹出文件选择窗口，请选中清单 Excel
    · 选错可关闭窗口后重新运行；下次会记住上次成功的清单

  第四步：查看预览并确认
    · 程序会先完成「分类预览」，显示：
        - 共处理多少个文件
        - 归入多少个项目、多少未分类
    · 询问「是否复制到已整理目录？」
        - 输入 Y 或直接回车 → 开始复制
        - 输入 n → 取消复制（待整理文件不变，可稍后再运行）

  第五步：查看结果
    · 确认后会自动打开「已整理/」文件夹
    · 同目录下会生成「整理结果-日期时间.txt / .xlsx」，可发给同事或留档
    · 未识别的文件在「已整理/_未分类/」

--------------------------------------------------------------------------------
【整理结果在哪里】

  · 已整理/<项目名称>/文件名          已归入项目的文件
  · 已整理/_未分类/文件名             未能识别的文件
  · 整理结果-20260609-1430.xlsx       本次整理明细（Excel，推荐查看）
  · 整理结果-20260609-1430.txt        同上（纯文本）
  · filekind-work/…/                  详细技术记录（一般不必打开）

  说明：工具是「复制」不是「移动」，待整理里的原文件始终保留。

--------------------------------------------------------------------------------
【项目清单说明（不必改配置）】

  · 不必在 projects.yaml 里填写清单文件名
  · 程序会在「待整理/」「项目清单/」中自动寻找像项目清单的 Excel
  · 不会把文件夹里每一个 Excel 都当作清单——只有表头正确的才会列入候选
  · 业务用的普通 Excel（如测试数据表、发票明细）不会被误当成清单

  若清单更新了：放入新版本 Excel，重新运行；多份并存时会提示您选择。

--------------------------------------------------------------------------------
【可选：调整 AI 分类说明（高级，可不填）】

  大多数文件靠「文件名 + 项目清单」即可分好类。
  仅当配置了本地 AI 模型时，才使用提示词文件：

  · 复制 classify_prompts.example.txt 为 classify_prompts.txt
  · 用记事本打开修改中文说明即可（纯文本，不是 YAML/JSON）
  · 不创建该文件也不影响日常使用

--------------------------------------------------------------------------------
【常见问题】

  问：待整理是空的，提示无法运行？
  答：请先放入至少一个待整理文件（清单 Excel 本身不算待整理文件）。

  问：很多文件进了「未分类」？
  答：1）确认项目清单 Excel 正确且已被选中；
       2）文件名是否含项目编号（如 202301_）或机型代号；
       3）查看「整理结果.xlsx」中「说明」列了解原因。

  问：会不会删我的原文件？
  答：不会。只复制到「已整理」，待整理原文件保留。

  问：整理错了怎么办？
  答：直接删除「已整理」里误复制的文件即可；待整理未改动。
       技术人员可用 manifest.json 做批量撤销（见 README）。

  问：一次能处理多少文件？
  答：默认最多 500 个/次。超出请分批放入待整理再运行。

  问：需要联网吗？
  答：分类过程不上传您的资料文件。首次运行需短暂联网下载本地模型
       （详见下文「Windows 首次联网说明」），之后同一台电脑可离线使用。

--------------------------------------------------------------------------------
【Windows 用户】

  1. 解压整个 filekind 文件夹（须含 filekind.exe 与 _internal 目录）
  2. 双击 run-filekind.bat（与 Mac 上双击 .command 相同）
  3. 步骤：待整理 → 选清单 → 预览确认 → 自动打开已整理

  若 bat 无法运行，可在该文件夹地址栏输入 cmd 回车，再执行：
      filekind.exe run --apply --no-dry-run --confirm --open-dest

  Windows 若提示「无法识别应用」：右键 run-filekind.bat → 以管理员身份运行，
  或在「更多信息」中选择仍要运行（视公司安全策略而定）。

--------------------------------------------------------------------------------
【Windows 首次联网说明】

  打包发给您的 zip 里主要是程序和运行库，并不包含全部 AI 模型文件。
  因此 Windows 电脑第一次整理时，通常需要联网一次（不上传您的业务文件）。

  ┌─────────────────────────────────────────────────────────────────────┐
  │ 1. 语义向量模型（几乎每台新电脑都会遇到）                              │
  │    · 何时下载：第一次运行整理，出现「加载向量模型…」时                  │
  │    · 大小：约 100 MB                                                 │
  │    · 保存位置：用户目录下（如 C:\Users\您的用户名\.cache\）           │
  │    · 作用：提高文件名/正文与项目的匹配准确率                            │
  │    · 没网时：仍可靠「项目清单 + 文件名规则」整理，但未分类可能偏多      │
  │    · 同一台电脑下载成功后，以后一般无需再下                             │
  ├─────────────────────────────────────────────────────────────────────┤
  │ 2. OCR 识别模型（仅有扫描版 PDF/图片时才需要）                        │
  │    · 何时下载：第一次处理扫出来的 PDF 或图片（无法直接复制文字）时      │
  │    · 保存位置：如 C:\Users\您的用户名\.paddleocr\                     │
  │    · 若资料都是 Word、可复制文字的 PDF：可能永远不会触发下载            │
  ├─────────────────────────────────────────────────────────────────────┤
  │ 3. 本地大语言模型 LLM（可选，默认可不装）                             │
  │    · 程序不会自动下载 LLM 权重文件                                     │
  │    · 若 zip 里没有 models\*.gguf，LLM 会自动跳过，不影响文员日常使用  │
  │    · 需要 LLM 时由 IT 预先放入 models 文件夹后再分发                   │
  └─────────────────────────────────────────────────────────────────────┘

  文员操作建议：
    · 第一次整理前：连接公司网络或 Wi-Fi，等待窗口不再长时间卡在「加载向量模型」
    · 若内网长期不能上网：请联系 IT 在一台可联网电脑上先运行一次整理，
      再将模型缓存目录复制到新电脑（或由 IT 统一预装）

  说明：下载的是公开模型文件，不是把待整理资料上传到互联网。
