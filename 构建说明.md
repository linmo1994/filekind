# filekind 构建说明

面向开发/打包人员。文员请看 `使用说明.txt`。

---

## 一、提示词文件在哪里

LLM 分类提示词（仅当 `projects.yaml` 中 `models.llm_gguf` 指向本地 `.gguf` 时才生效）：

| 文件 | 作用 | 位置 |
|------|------|------|
| `classify_prompts.txt` | **实际使用**的提示词（纯文本，推荐） | 与 `projects.yaml` 同目录（开发时在仓库根目录，打包后在 `dist/filekind/`） |
| `classify_prompts.example.txt` | 纯文本模板，复制为 `classify_prompts.txt` 后修改 | 仓库根目录 / `dist/filekind/` |
| `classify_prompts.example.yaml` | 旧版 YAML 模板（仍兼容） | 仓库根目录 / `dist/filekind/` |
| 内置默认提示词 | 找不到上述文件时使用 | 代码内：`filekind/prompts.py`（`DEFAULT_MERGED_SYSTEM` / `DEFAULT_MERGED_USER`） |

加载优先级（见 `filekind/prompts.py`）：

1. `projects.yaml` 的 `paths.classify_prompts` 指定的文件（默认 `classify_prompts.txt`）
2. 同目录下的 `classify_prompts.txt` → `classify_prompts.example.txt`
3. `classify_prompts.yaml` → `classify_prompts.example.yaml`
4. 都没有则用 `filekind/prompts.py` 的内置默认值

纯文本格式用两个分节标记：

```
=== 系统说明 ===
（业务/分类规则，可改）

=== 用户模板 ===
文件名：{filename}
...（保留 {占位符}，勿删大括号）
```

> 占位符如 `{filename}`、`{candidate_projects_json}` 等由程序自动替换；不存在该文件不影响规则 + 向量分类，只影响 LLM 阶段。

## 二、Windows 构建

> 注意：**不能**把 macOS 上打好的 `dist/filekind/` 拷到 Windows 使用。必须在 **Windows 本机**（或 GitHub Actions 的 windows runner）重新打包，产物为 `filekind.exe`。

### 0. 环境要求

- Windows 10/11
- Python 3.11（与 CI 一致）
- 可访问 PyPI（安装依赖时联网）

### 1. 拷贝到 Windows 的内容

推荐用 Git 拉取整个仓库：

```bat
git clone <仓库地址> filekind
cd filekind
```

若手动拷贝，至少包含：

**必须**
- `filekind/`（主程序源码）
- `filekind_main.py`（打包入口）
- `filekind.spec`（PyInstaller 配置）
- `pyproject.toml`（依赖定义）
- `scripts/`（`bundle_dist.py`、`run-filekind.bat`、`smoke_test_build.py`、`pyi_rth_paddle.py`）
- `projects.example.yaml`
- `classify_prompts.example.txt`、`classify_prompts.example.yaml`
- `使用说明.txt`

**可选**
- `tests/`（构建前自测）
- `models/*.gguf`（若要在发布包内自带 LLM，约 1GB）
- `README.md`、`构建说明.md`

**不要拷贝**（在 Windows 上重新生成）
- `dist/`、`build/`（旧平台产物）
- `.venv/`、`venv/`（虚拟环境不可跨机复用）
- `__pycache__/`、`.pytest_cache/`、`filekind-work/`

### 2. 创建虚拟环境并安装依赖

```bat
python -m venv .venv
.venv\Scripts\activate
python -m pip install --upgrade pip setuptools wheel
pip install -e ".[dev,build,llm,ocr]"
```

依赖分组（`pyproject.toml`）：
- `build` → PyInstaller（必需）
- `llm` → 本地大模型推理 llama-cpp（可选，体积大）
- `ocr` → PaddleOCR，用于扫描版 PDF/图片（可选，依赖多、体积大）

仅需「规则 + 向量」时可精简：

```bat
pip install -e ".[dev,build]"
```

### 3. 打包

```bat
pyinstaller --noconfirm --clean filekind.spec
python scripts\bundle_dist.py
```

- `pyinstaller` 生成 `dist\filekind\filekind.exe` 与 `_internal\`
- `bundle_dist.py` 复制配置模板、`使用说明.txt`、`run-filekind.bat`，并创建 `待整理/`、`已整理/`、`项目清单/`

### 4. 自检（可选）

```bat
python scripts\smoke_test_build.py windows
```

### 5. 可选：把 LLM 打进发布包

```bat
python scripts\download_llm_model.py
python scripts\bundle_dist.py
```

第二次 `bundle_dist.py` 会把 `models\*.gguf` 纳入 `dist\filekind\models\`。
不带 LLM 时，请将 `projects.yaml` 的 `models.llm_gguf` 留空（程序会自动跳过 LLM）。

### 6. 发布给文员

发送 **整个文件夹**（不要只发 exe）：

```
dist\filekind\
├── filekind.exe
├── _internal\          ← 必须随 exe 一起分发
├── run-filekind.bat    ← 文员双击入口
├── projects.yaml
├── 使用说明.txt
├── classify_prompts.example.txt
├── 待整理\
├── 已整理\
└── 项目清单\
```

打包为 zip（与 CI 一致）：

```powershell
tar -a -c -f filekind-windows.zip -C dist filekind
```

## 三、macOS / Linux 构建（对照）

```bash
chmod +x scripts/build_executable.sh
./scripts/build_executable.sh
```

- 产物：`dist/filekind/filekind`
- 文员入口：`双击可整理文件.command`
- OCR：优先使用系统 Vision；其他平台用 PaddleOCR

---

## 四、用 GitHub Actions 免本机构建

仓库已配置 `.github/workflows/ci.yml`：推送到 `main` 后，
在 Actions 中的 **Build (windows)** 任务下载制品 `filekind-windows.zip`，
解压后即为 `dist/filekind/`，可直接发给用户。

## 五、常见问题

| 问题 | 处理 |
|------|------|
| `llama-cpp-python` 安装失败 | 改用 `pip install -e ".[dev,build,ocr]"` 跳过 LLM；文员包通常无需 LLM |
| `paddlepaddle` 安装慢/失败 | 改用 `pip install -e ".[dev,build]"`；无 OCR 时仅扫描件识别受影响 |
| 文员电脑要不要装 Python | 不需要，只需 `dist\filekind\` 整个文件夹 |
| 首次运行是否联网 | 需要，详见 `使用说明.txt`「Windows 首次联网说明」（约 100MB 向量模型；OCR 仅扫描件时下载；LLM 不自动下载） |

## 六、最短命令清单（Windows 构建机）

```bat
git clone <repo> filekind && cd filekind
python -m venv .venv && .venv\Scripts\activate
pip install -e ".[dev,build,llm,ocr]"
pyinstaller --noconfirm --clean filekind.spec
python scripts\bundle_dist.py
```

产物：`dist\filekind\`，连同 `使用说明.txt` 一起 zip 发给文员。
